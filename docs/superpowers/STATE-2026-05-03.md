## What's NOT WORKING (current blocker)

### Bug: ExitInterior cannot escape interior maps

Symptom in V2.4.5 run at `Indoors(mapId=0, localX=7, localY=6)`:
- `findPathToExit` returns "BLOCKED. no exit (DOOR/STAIRS/WARP or south-edge) visible in viewport"
- `exitInterior` fails immediately — pathfinder doesn't even start walking.

Root cause: **viewport-vs-full-map mismatch.**

`InteriorPathfinder.findPath` operates on a `ViewportMap` (16×16 around party).
When party spawns at edge of the map (here localX=7, localY=6 — top-left
corner), the viewport extends from (-1, -2) to (14, 13). The ACTUAL exit (south
edge of playable area) is around localY=14-15 — **outside the visible window**.

Earlier runs (V2.4.1 / V2.4.4) had party at `(5, 28)` deep enough into the map
that the viewport DID see south-edge exits, and findPath returned valid 17-step
paths. Those runs failed on a different bug (button-press timing — fixed by
bumping FRAMES_PER_TILE to 48 in V2.4.5).

V2.4.5 changed the failure mode but didn't reveal whether the timing fix
actually works, because party never spawned where viewport covered the exit.

## The fix (proposed for next session — V2.4.6)

`InteriorPathfinder` should BFS the **full 64×64 InteriorMap** instead of a 16×16
viewport. Interior maps are tiny (~4KB each), full BFS is microseconds. We
already have `MapSession.currentMap` exposing the InteriorMap directly.

Two implementation paths:

**A) Full-map pathfinder, signature change.**
Add `InteriorMap` parameter to InteriorPathfinder.findPath; ExitInterior
passes `mapSession.currentMap` instead of viewport. ViewportMap-based BFS
becomes a degraded fallback (only used if MapSession not configured).

**B) Reshape readViewport to return wide enough region around party.**
`InteriorMap.readViewport` currently emits 16×16 around party. Bump it to
64×64 (whole map). Pathfinder code unchanged. ViewportMap renderer for
advisor should still display only the 16×16 around party (to keep prompt
size sane); add a separate `readMapView()` for pathfinder.

**Recommendation: B**, less invasive. Adds an `InteriorMap.readFullMapView()`
returning a 64×64 ViewportMap; pathfinder uses that, advisor's `readViewport()`
unchanged.
